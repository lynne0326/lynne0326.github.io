Search Engine Review I

发表于 2017-02-27

Introduction:
Search Engine 用于IR information retrieve, 这一过程包含将用户的information need 进行一系列加工最后得到需要的result返回给用户，目的是为了find documents satisfy information need.

Concepts:
Information need: 用户想要得到的信息需求。
Query:是一种approximate information need的表现方式。
Documents: corpus of unstructured information，非结构化的文体。

Extraction 的流程里第一步：

1.Represent information need 表示IN的方式

Exact match model：和名字差不多，就是yes or no的形式，有就是有，没有就是没有，典型的例子就是database，eg. A AND B, A OR B, A NEAR/2 B (A和B之间隔了1个词 distance = ib- ia, 等于0没有意义)

Buld Query Tree: 将query表现成树的形式，leaves都是terms，leaves就是指针指向inverted list在disk中的位置。

2.Represent Document

Free-text indexing: 用doc中的一些词来表示这个doc。
Full-text indexing: 用所有或者大多数词(most or all words)来表示这个doc。
对于Full-text indexing, 又有几种不同的表示方式：

The Binary Full-Text Representation(The Bag of Words), 用一个表来纪录某个单词是否出现在某个doc里，出现为1不出现为0.

另一种是Frequency-Based Full-Text Representation, 和前面的表相似，但是数值对应某个单词出现在某doc里的frequency而不是1或0.

Retrieval Model:
Ranked:按照score排序
Unranked: 出现即score1，不出现则0，没有排序
